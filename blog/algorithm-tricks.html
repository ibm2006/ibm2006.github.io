Algorithm / Tricks

Initial notes

This page collects small algorithmic ideas that are useful in competitive programming. The goal is not to replace full editorials, but to keep a compact list of reusable patterns and warning signs.

Tricks to Organize

  • Prefix sums, difference arrays, and offline range processing.
  • Two pointers, sliding windows, and monotonic queues.
  • Coordinate compression and sweep line arguments.
  • Small-to-large merging and maintaining data while traversing a tree.
  • Binary search on the answer with a clean feasibility check.

Things to Watch

  • Whether a greedy choice has an exchange argument.
  • Whether the state in a dynamic programming solution is too large.
  • Whether sorting changes the structure enough to simplify the problem.
  • Whether the same contribution can be counted from each element's viewpoint.
